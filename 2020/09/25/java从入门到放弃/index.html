java简介

Java介于编译型语言和解释型语言之间。在平台和源码之间多了一层虚拟机。

常用的版本为java SE，现在的java版本为15.

所谓JDK和JRE

  • JDK：Java Development Kit
  • JRE：Java Runtime Environment

简单地说，JRE就是运行Java字节码的虚拟机。但是，如果只有Java源码，要编译成Java字节码，就需要JDK，因为JDK除了包含JRE，还提供了编译器、调试器等开发工具。

基本结构

1、定义类（class）

类名要求：

  • 类名必须以英文字母开头，后接字母，数字和下划线的组合
  • 习惯以大写字母开头

注意到public是访问修饰符，表示该class是公开的。不写public，也能正确编译，但是这个类将无法从命令行执行。

变量定义

几乎与C一样，下面列举区别

java有引用类型，引用类型最常用的就是String字符串：

引用类型的变量类似于C语言的指针，它内部存储一个“地址”，指向某个对象在内存的位置。

Java的编译器对字符串做了特殊照顾，可以使用+连接任意字符串和其他数据类型，这样极大地方便了字符串的处理。

Java的字符串除了是一个引用类型外，还有个重要特点，就是字符串不可变。这里指的是字符串的内容不可变，但指向可变。

判断引用类型的变量是否相等，==表示“引用是否相等”，或者说，是否指向同一个对象。要判断引用类型的变量内容是否相等，必须使用equals()方法：

流程控制

输出

可以使用System.out.println()来向屏幕输出一些内容。println是print line的缩写，表示输出并换行。因此，如果输出后不想换行，可以用print()：

如果要把数据显示成我们期望的格式，就需要使用格式化输出的功能。格式化输出使用System.out.printf()，通过使用占位符%?，printf()可以把后面的参数格式化成指定格式，占位符与c一致

输入

通过import语句导入java.util.Scanner，创建Scanner对象并传入System.in。有了Scanner对象后，要读取用户输入的字符串，使用scanner.nextLine()，要读取用户输入的整数，使用scanner.nextInt()。

switch

基本使用与C一致

使用switch时，注意case语句并没有花括号{}，而且，case语句具有“穿透性”，漏写break将导致意想不到的结果。

另一种新语法

注意新语法使用->，如果有多条语句，需要用{}括起来。不要写break语句，因为新语法只会执行匹配的语句，没有穿透效应。
